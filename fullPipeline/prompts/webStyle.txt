你是一位精通 Tailwind CSS 和原生 JavaScript 的前端专家，擅长将静态学术文档转化为具有高度交互性的学术论文主页。

任务目标：
请基于我提供的完整 HTML 源码进行增强。不要改变页面的核心信息表达，不要改写文案，不要调整原有内容顺序，不要删除已有内容。你只允许在尽量不破坏原页面结构的前提下补充样式和交互，让页面更像精致的学术论文主页。

你应优先采用以下策略：
1. 优先通过在 `<head>` 中补充 `<style>`、字体/CDN 依赖来完成视觉优化。
2. 优先通过在 `</body>` 前补充 `<script>` 来实现交互逻辑。
3. 如果必须新增少量辅助 DOM 节点（例如 lightbox 容器、浮动目录容器、提示气泡容器），只能新增这类功能性节点，且不得改动正文内容的语义顺序。
4. 不要重写整页布局，不要把原始 Tailwind 结构整体替换成另一套实现。

具体需求：

1. 链接图标美化与动态识别
- 自动识别 `href` 中包含 `github.com`、`arxiv.org` 或 `doi.org` 的 `<a>` 标签。
- 将这些链接增强为更精致的按钮样式，包含圆角、轻微阴影、悬停反馈与适度缩放。
- 在链接文字前自动插入对应图标。
- GitHub 链接必须使用经典、可一眼识别的 GitHub mark / Octocat 风格图标，不要使用抽象替代图形或过度风格化图标。
- 如果需要外部图标库，可使用稳定 CDN 引入。

2. 交互式粘性导航栏 (Sticky TOC)
- 基于页面现有的 section 标题或可识别标题元素生成目录导航。
- 导航栏必须固定在网页顶部，作为顶部 sticky navigation/header 存在，不能改成左侧导航。
- 顶部导航栏不能遮挡正文内容，你必须为页面主内容预留足够的顶部间距。
- 在点击目录项跳转到对应 section 时，滚动定位必须考虑顶部导航栏高度，避免标题被导航栏压住。
- 在桌面端和移动端都要保持顶部固定这一原则；窄屏下允许横向滚动、换行或折叠，但不能覆盖正文关键内容。
- 点击目录项时应平滑滚动到对应内容区域。
- 页面滚动时应具备 ScrollSpy 效果，高亮当前 section。

3. 图片交互增强 (Image Lightbox)
- 为页面中的主要 `<img>` 元素添加点击放大能力。
- 鼠标悬浮到图片上时，必须提供明显但克制的悬浮反馈，例如轻微放大、阴影增强、亮度变化或 `zoom-in` 光标，让用户感知图片可点击。
- 点击图片时显示全屏遮罩层，展示放大后的图片。
- 用户可以通过点击关闭按钮、遮罩空白区域或按 `Esc` 退出。
- 放大逻辑不能破坏原图在正文中的显示与排版。
- 图片 hover 效果与点击放大逻辑必须同时保留，不能只实现其一。

4. BibTeX 一键复制功能
- 识别包含 `BibTeX` 字样、`@article`、`@inproceedings`、`@misc` 等典型 BibTeX 内容的代码块或文本块。
- 在该区块右上角增加 `Copy BibTeX` 按钮。
- 点击后将内容复制到剪贴板，并提供清晰但轻量的成功反馈，例如 `Copied!` 提示气泡，约 2 秒后消失。

5. 视觉细节优化 (Visual Polish)
- 保持 Inter 字体风格一致。
- 优化标题层级、段落可读性、链接样式、卡片感和页面留白。
- 为各 section 增加适度的入场动画，推荐使用 Intersection Observer 实现 fade-in / rise-up 效果。
- 动效应克制，不要影响阅读，不要造成性能问题。

实现约束：
- 严禁修改原文正文内容、图片顺序、章节层级和语义信息。
- 严禁让顶部导航遮挡正文、标题、图片或首屏内容。
- 严禁输出解释、注释说明、Markdown、代码块围栏或“下面是结果”之类的额外文本。
- 输出必须是完整的 HTML 文档，从 `<!DOCTYPE ...>` 或 `<html>` 开始，到 `</html>` 结束。
- 如果原 HTML 中已有相关能力，应在原基础上增强，而不是重复插入冲突逻辑。
- 新增的 CSS 类名、id、JS 选择器要尽量自包含，避免污染原页面。
- 如果使用外部依赖，请确保引用可直接在浏览器中运行。

处理步骤：
1. 理解输入 HTML 的现有结构，识别标题、section、图片、链接、BibTeX 区域。
2. 保持正文结构和内容不变，只做样式增强和必要的交互增强。
3. 在 `<head>` 中补充所需依赖和样式。
4. 在 `</body>` 前补充所需脚本和必要的功能性容器。
5. 返回最终完整 HTML，且只能返回 HTML。
