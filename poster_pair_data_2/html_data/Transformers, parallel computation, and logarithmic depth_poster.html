Transformers, Parallel Computation, and Logarithmic Depth

Clayton Sanford, Daniel Hsu, Matus Telgarsky

Columbia University
New York University
Spotlight Paper

Motivation

Motivating question: How can transformers be understood as models of parallel computation (e.g. MPC)?

  • Representational powers of depth?
  • Multi-step reasoning capabilities?
  • Comparisons with other models (state-space models, sub-quadratic attention transformers, GNNs)?

Transformer Architecture

  • Self-attention unit: f(X) = softmax(XQKᵀXᵀ)XV
  • Multi-headed attention: L(X) = X + Σ h_i(X)
  • Arbitrary element-wise encoding (MLP + positional encoding)
  • Full transformer: T(X) = (φ_D ∘ L_D ∘ ... ∘ L_1 ∘ φ_0)(X)

Massively Parallel Computation (MPC)

Theoretical model of MapReduce by Karloff, et al '10.

  • N-bit input divided among q machines with local memory s.
  • For round r = 1, ..., R: Each machine performs computation and communicates.

(γ, δ)-MPC protocols have local memory s = O(N^δ) and global memory qs = O(N^{1+γ}).

Conjecture (1-cycle vs 2-cycle; Ghaffari, et al '19)

Any (γ, δ)-MPC protocol with δ ∈ (0, 1) that distinguishes one length-N cycle from two length-N/2 cycles uses R = Ω(log N) rounds.

k-Hop Induction Heads

Induction Heads: Toy task to complete bigram via previous occurrence: IH(baebcabebdea) = b.

  • Occurs naturally in trained transformers; useful primitive for mechanistic interpretability.
  • Two-step logical reasoning task with natural two-layer transformer construction.

k-hop Induction Heads: Iterated variant of IH to capture k-step reasoning.

Theory Result #1: Near-equivalence of Transformers & MPC

Theorem (Transformers simulate MPC)

Any R-round (γ, δ)-MPC protocol can be simulated by a transformer of depth L = O(R) and width m = Õ(N^δ).

Theorem (MPC simulates Transformers)

Any transformer of depth L and width m can be simulated by an MPC protocol with R = O(L) rounds.

Theory Result #2: Efficient Multi-Step Reasoning Representations

Construction: Transformers of depth L = ⌊log₂ k⌋ + 2 and width m = O(1) can solve k-IH.

  • First two layers solve induction heads for each input.
  • Subsequent layers employ "pointer doubling" to chain together solutions.

Theory Result #3: Sub-optimality of Other Architectures

Graph Neural Networks: Solving graph connectivity requires L√m = Õ(N^{1/4}).

State-Space Models: RNN, LSTM, Mamba solutions to k-IH require depth L ≥ k or width m = Ω(N/k⁶).

Sub-Quadratic Attention Transformers: Every kernel-based or masking-based transformer variant that solves k-IH has either depth L ≥ k or near-quadratic computation.

Empirical Result #1: Log-depth k-Hop Induction Heads

Task: Multi-task k-IH for k = 0, ..., 16; sequence length N = 100.

Result: Low error when L ≤ ⌊log₂ k⌋, as predicted by theory.

Empirical Result #2: Interpretability of Learned Models

16-IH attention matrices with colored partial solutions.

Yes. Close correspondence between self-attention matrices and partial solutions to k-IH.
