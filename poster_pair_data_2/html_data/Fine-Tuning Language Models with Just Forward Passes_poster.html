Fine-tuning LLMs with backpropagation requires a prohibitively large amount of memory.
We propose MeZO, a memory-efficient zeroth-order optimizer.
Costs the same memory as inference without sacrificing much performance.
With one A100, MeZO can tune a 30B model while Adam can only tune a 2.7B model.

MeZO Algorithm

Classic zeroth-order (ZO) gradient estimate with two forward passes: SPSA [1]

∇L(θ; B) ≈ (L(θ + εz; B) - L(θ - εz; B)) / (2ε) * z

Memory-efficient ZO-SGD (MeZO): resample z so SGD with SPSA uses no extra memory besides inference cost.

(by first sampling a random seed and later restoring the random seed to make sure the sampled z is consistent)

Why does it work?

Classical results: ZO slows down by number of parameters d.

Our case: fine-tuning a pre-trained model with a prompt, so fine-tuning looks like pre-training!

Then: effective rank of Hessian r is small, convergence rates depend on r instead of d (descent lemma).

Experiments

Our examples cover medium-sized masked LMs (RoBERTa-large) and large autoregressive LMs (OPT-13B, 30B, 66B).

Both full-parameter and parameter-efficient fine-tuning (e.g., LoRA, prefix-tuning).

We experiment on classification, multiple-choice, and generation tasks.

  • 1. On RoBERTa-large, MeZO outperforms zero-shot and linear probing.
  • 2. MeZO is compatible with both full-parameter and parameter-efficient fine-tuning (PEFT), such as LoRA [2], prefix-tuning [3].
  • 3. On OPT-13B, MeZO achieves on par performance with Adam on 7/11 tasks, while saving up to 12x memory.
  • 4. MeZO also beats memory-equivalent methods like in-context learning and linear probing.

5. MeZO can scale up to 66B models.

6. MeZO works with non-differentiable objectives.

When should you use MeZO

  • efficient, fuzzy gradient estimate (e.g., data selection)
  • optimize non-differentiable objectives (e.g., with a reward model)
  • study huge models but don't have many GPUs

When shouldn't you use MeZO

  • Does fine-tuning a model do X?
  • I have a new task without obvious prompts
  • I only want to study small models
