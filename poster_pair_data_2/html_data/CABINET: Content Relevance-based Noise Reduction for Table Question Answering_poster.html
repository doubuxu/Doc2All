CABINET: Content Relevance Based Noise Reduction for Table Question Answering

Sohan Patnaik, Heril Changwal, Milan Aggarwal, Sumit Bhatia, Yaman Kumar, Balaji Krishnamurthy

Media and Data Science Research Lab, Adobe, India
github.com/adobe-research/CABINET

LLMs have been used for Table Understanding

Large Tabular Data degrades LLM Performance

Hard Decomposition can cause Information Loss

Our Approach: CABINET assigns Relevance Score to Tabular Data Tokens through Two Modules - 1) Unsupervised Relevance Scorer and 2) Weakly Supervised Relevant Cell Highlighter

CABINET (despite having less #params) outperforms several SoTA Methods

CABINET is more Robust to Noise and Table Perturbations

RA – Row Addition

RP – Row Permutation

CP – Column Permutation

CR – Cell Replacement

CABINET is more Performant while Handling Irrelevant Data in Large Tables

Clustering URS Output Representations is Helpful

Ablation Study

Using Both Modules gives Best Performance
